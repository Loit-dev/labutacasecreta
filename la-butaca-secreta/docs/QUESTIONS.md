# Motor de Preguntas

## Objetivo

Descubrir qué necesita realmente el usuario haciendo el menor número posible de preguntas.

Nunca se harán preguntas innecesarias.

---

# Principios

Cada pregunta debe aportar información.

Si una respuesta ya permite deducir otra, esa segunda pregunta nunca se mostrará.

---

# Prioridades

1. Tipo de contenido.
2. Compañía.
3. Estado de ánimo.
4. Intensidad.
5. Ritmo.
6. Duración.
7. Restricciones.
8. Afinado final.

---

# Finalización

El flujo termina cuando el nivel de confianza supera el umbral definido por el motor.

No existe un número fijo de preguntas.

El máximo será diez.

El mínimo dependerá de la confianza alcanzada.

---

# Filosofía

Cada conversación debe sentirse diferente.

Nunca debe parecer un formulario.

---

## Regla de Oro

Ninguna pregunta puede ignorar las respuestas anteriores.

Toda pregunta debe adaptarse al contexto modificando al menos uno de estos elementos:

- texto;
- opciones;
- orden de las opciones;
- mensaje introductorio;
- o decidir no mostrarse.

Si una pregunta no cambia en función del contexto, probablemente deba replantearse.
